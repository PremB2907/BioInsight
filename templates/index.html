SYSTEM BIO-INFORMATICS & DATA MINING WORKFLOW

1. PREPROCESSING & MIN-MAX NORMALIZATION

The leukemia dataset represents high-dimensional expression counts of 1,000 genes across 1,000 samples. To eliminate scaling bias, missing rows are dropped, and columns are normalized to a strict `[0, 1]` index scale:

X_norm = (X - X_min) / (X_max - X_min)

This mirrors the **WEKA Normalize** unsupervised filter (`scale = 1.0, translate = 0.0`), preventing high-range variance genes from distorting distance coordinates in K-Means.

2. FEATURE SELECTION (ANOVA F-SCORE)

To solve the *curse of dimensionality* (1,000 genes/dimensions), we use **ANOVA F-score ranking** (mirroring WEKA's Attribute Selection evaluator with Ranker). This metrics computes:

F = (Variance between classes) / (Variance within classes)

We rank the genes by F-score and select the top 20 genes that explain the most statistically significant variance between classes (**ALL**, **AML**, and **Healthy**).

3. UNSUPERVISED CLUSTERING (K-MEANS)

The normalized dataset is clustered into $K=3$ partitions using SimpleKMeans (Euclidean distance). This allows us to observe if unsupervised geometric groupings represent the actual clinical diagnosis labels.

**The Cluster Alignment challenge:** Unsupervised models do not know the biological context of the groupings. They label them arbitrarily (e.g. `cluster0`, `cluster1`). To align them, we run classification accuracies for all $3! = 6$ possible label permutations to discover the optimal cluster mapping.
